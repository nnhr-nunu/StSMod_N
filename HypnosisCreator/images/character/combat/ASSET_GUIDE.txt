# バトル立ち絵・モーション素材

戦闘中のキャラ表示用PNG。PNGだけでは動かず、
scenes/creature_visuals/ の .tscn から参照します。

## 配置するファイル

| ファイル名（推奨） | 用途 | 必須 |
|--------------------|------|------|
| idle.png           | 待機（1枚フォールバック） | ○（単枚時） |
| idle/frame_000001.png 〜 | 待機（idle_loop） | 任意 |
| damage/frame_000001.png 〜 | 被弾（hurt） | 任意 |
| power/frame_000001.png 〜 | 詠唱／パワー（cast） | 任意 |
| attack/frame_000001.png 〜 | 攻撃（attack） | 任意（768×1152 に揃える） |
| die.png            | 死亡 | 任意 |

最低限 idle.png、または idle/ の連番フレームがあれば表示できます。
連番は scenes/creature_visuals/combat_frames.tres にまとめ、
ゲームのアニメリクエスト（被弾・詠唱など）で切り替わります。

## ウォーターマーク（全モーション共通）

Hailuo / MiniMax 系の白文字ロゴはだいたい同じ右下位置に出ます。

- **PNG は改変しない**（靴などを壊しやすいため）
- 戦闘立ち絵は `shaders/chroma_key`（`chroma_key_material.tres`）を共有
- シェーダー側の角抜き（`wm_crop_*`）＋白いロゴ文字検出で透明化
- idle / damage / attack など、同じ素材規格なら **そのまま使い回し**
- Visuals は Flip H = ON 前提（ロゴが UV 左側に来る）

位置がズレた／残るときは Mod設定 → Watermark スライダー
（CropBottom / CropSide / LogoStrength）で調整。
敗北は die 連番なしでシェーダー・ディゾルブ消滅（CombatDeathDissolve）。
被弾はほぼ静止の序盤フレームを飛ばし、`hurt` を高速再生している。

## 仕様

- 拡張子: .png（透過推奨）
- **統一キャンバス: 768×1152**（idle / damage / power / attack すべて）
  - 生成元が 1080p などでも、取り込み時にこのサイズへ揃える
  - 構図の余白比率が近いほど、モーション切替でサイズ差が出にくい
- 背景: グリーンバック（GB）のままでOK
  - 戦闘シーンは shaders/chroma_key で緑を透過
  - 手で透過PNGにしてもそのまま使える（二重でも害は少ない）
- 向き: ゲーム内では「敵側（画面右）を向く」のが標準
  - 素材が左向きの場合 → Godot の Sprite2D / AnimatedSprite2D で Flip H = ON
  - または scale.x = -1

## 命名ルール

- 小文字 + アンダースコア
- フォルダ連番: frame_000001.png 〜
- 例: idle/frame_000001.png / damage/frame_000001.png

## 次の作業（実装側）

1. モーション連番を images/character/combat/<motion>/ に置く
2. SpriteFrames（または Animation）を scenes/creature_visuals/ に追加
3. Visuals に chroma_key_material を割り当て（WM隠し込み）
4. dotnet publish
